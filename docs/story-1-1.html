Story 1.1: Project Initialization & Design System

Status: ready-for-dev

Story

As a développeur Tiba,
I want un mono-repo correctement configuré avec le Design System Tiba implémenté,
So that toutes les futures stories s'appuient sur une fondation technique et visuelle cohérente.

Acceptance Criteria

  1. Given le développeur clone le dépôt pour la première fois
    When il exécute les commandes d'initialisation
    Then la structure mono-repo contient apps/tiba-mobile/ (Flutter), apps/tiba-admin/ (Next.js stub), et supabase/
    And le projet Flutter est initialisé via flutter create --org com.tiba avec le package name com.tiba.app
  2. Given le projet Flutter est initialisé
    When le développeur ouvre lib/core/theme/
    Then le fichier tiba_theme.dart contient le ThemeData Material 3 surchargé avec :
    • Couleur primaire : Vert Santé #0B8E71
    • Couleur de fond : Sable Chaud #F5F0E6
    • Couleur d'alerte : Rouge Corail #E85D75
    • Couleur texte : Charbon Profond #1A1A1A
    • Border radius global : 24px (soft edge)
    • Polices : Outfit (titres), Inter (corps de texte)
      And tous les boutons et cibles interactives ont un minimum de 48x48px (UX-DR6)
      And le ratio de contraste texte/fond respecte 7:1 minimum (NFR20)
  3. Given le Design System est en place
    When le développeur navigue dans l'application
    Then la Bottom Navigation Bar de 3 onglets (Scanner, Historique, Secours) est visible (UX-DR4)
    And aucun Menu Hamburger n'existe dans l'application
  4. Given le projet est configuré
    When flutter analyze est exécuté
    Then zéro erreur et zéro warning sont reportés

Tasks / Subtasks

  • Task 1: Initialize Flutter Mobile App (AC: #1)
    • Initialize standard Flutter project tiba_app with org com.tiba within tiba-workspace/apps/tiba-mobile/.
    • Configure Android and iOS application ids to com.tiba.app.
    • Remove all boilerplate code from main.dart.
  • Task 2: Implement Tiba Design System (AC: #2)
    • Import Outfit and Inter Google Fonts.
    • Create lib/core/theme/tiba_theme.dart and define the central ThemeData.
    • Implement exact color codes (#0B8E71, #F5F0E6, #E85D75, #1A1A1A) and 24px shapes.
    • Create a reusable TibaButton widget ensuring minimum 48x48px touch targets.
  • Task 3: Setup Application Navigation Shell (AC: #3)
    • Implement a base Scaffold shell with a BottomNavigationBar.
    • Add 3 tabs: "Scanner", "Historique", "Secours".
    • Ensure NO hamburger menu (Drawer) is present in the app architecture.
  • Task 4: Fix Linter and Tests (AC: #4)
    • Configure analysis_options.yaml for strict linting.
    • Run flutter analyze and resolve any issues.

Dev Notes

Project Structure Notes

  • Alignment: This is a brownfield project where the workspace is partially generated by the architect agent (tiba-workspace/). The developer MUST navigate inside tiba-workspace/apps/tiba-mobile/ before running Flutter commands.
  • State Management: Use Riverpod as per architecture rules.
  • Naming: Dart/Flutter uses snake_case.dart for files and PascalCase for classes.
  • Constraints: Max APK size is <30MB, keep dependencies and assets ultra-light.

Architecture Compliance

  • Technical Stack: Flutter 3.x (Dart).
  • Security: N/A for this story, but UI foundations must follow UX 7:1 contrast.
  • Testing Standards: Standard unit tests for theme values and widget tests for the navigation shell.
